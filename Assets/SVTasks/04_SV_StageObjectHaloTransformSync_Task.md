# Task 04. `StageObject_Halo` 위치/회전/크기 동기화

## 목적
`StageObject_Halo` 1개의 위치, 회전, 크기 값을 Host 권위로 동기화해 3클라이언트가 같은 월드 상태를 공유하게 만든다.

## 구현 대상
- 위치 동기화
- 회전 동기화
- 크기 동기화
- Host에서 값을 바꾸는 테스트용 입력 또는 디버그 컨트롤

## 구현 범위
- 위치와 회전은 Fusion 기본 동기화 컴포넌트를 활용할 수 있다.
- 크기는 기본 `NetworkTransform`만으로 충분하지 않을 수 있으므로 별도 `Networked` 값 또는 커스텀 동기화 로직을 고려한다.
- 상태 변경 권한은 Host만 가진다.
- 동기화 대상은 월드 기준 Transform이며, 관객 로컬 좌표 보정값은 넣지 않는다.

## 구현 단계
1. 현재 프로젝트의 Fusion 컴포넌트로 위치/회전 동기화가 어디까지 가능한지 확인한다.
2. 크기 값은 별도 네트워크 프로퍼티가 필요한지 결정한다.
3. Host에서 `StageObject_Halo`를 이동, 회전, 확대/축소할 수 있는 최소 테스트 입력을 만든다.
4. 클라이언트 3대에서 동일 프레임은 아니더라도 동일 상태로 수렴하는지 확인한다.
5. 접속 후반에 들어온 클라이언트도 최신 위치/회전/크기를 받는지 확인한다.

## 공연/AR 맥락 반영
- 공연 연출 오브젝트의 권위 값은 하나지만, 실제 보이는 각도는 관객 시점에 따라 달라진다.
- 따라서 이 Task는 월드 상태 일치에 집중하고, 시점 차이는 AR 카메라가 해결하도록 분리한다.

## 가정
- 연출 오브젝트는 과도하게 빠르게 움직이지 않는 프로토타입 속도를 사용한다.
- 스케일 값은 균일 스케일 또는 정의된 축 스케일 범위 안에서 먼저 검증한다.

## 검증 필요
- 스케일 동기화에 보간이 필요한지 여부를 실제 연출 속도로 확인해야 한다.
- 회전 표현이 Euler인지 Quaternion인지에 따라 디버그 UI 표시 방식 검토가 필요하다.

## 완료 기준
- Host에서 바꾼 위치, 회전, 크기가 모든 클라이언트에서 동일한 월드 값으로 반영된다.
- 늦게 접속한 클라이언트도 최신 상태를 받는다.
- 다음 Task에서 스마트폰 카메라 화면 위에 같은 오브젝트를 자연스럽게 보여줄 수 있다.
