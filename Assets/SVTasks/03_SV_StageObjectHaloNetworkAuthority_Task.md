# Task 03. `StageObject_Halo` 네트워크 오브젝트 권한화

## 목적
공연장 안의 단일 연출 오브젝트인 `StageObject_Halo`를 Host 권한 기반의 네트워크 오브젝트로 정리한다.

## 구현 대상
- `StageObject_Halo`의 `NetworkObject` 구성
- Host만 생성 또는 소유하도록 하는 권한 정책
- 씬 배치형 또는 런타임 Spawn 방식 중 하나의 기준 확정
- 네트워크 상의 식별 이름과 참조 경로 정리

## 구현 범위
- 동기화 대상은 `StageObject_Halo` 1개만 다룬다.
- 오브젝트 자체의 생성 책임은 Host에만 둔다.
- 클라이언트는 이 오브젝트를 수정하지 않고 수신/표시만 한다.
- 공연장 좌표계 기준 위치를 이 오브젝트의 권위 값으로 취급한다.

## 구현 단계
1. 현재 씬의 `StageObject_Halo`를 확인하고 네트워크 오브젝트화 전략을 정한다.
2. Scene Object로 둘지, Host Spawn Prefab으로 둘지 프로토타입에 더 단순한 방식을 선택한다.
3. Host만 해당 오브젝트를 활성화하거나 Spawn하도록 제한한다.
4. 클라이언트는 동일한 네트워크 오브젝트 참조를 수신하는지 확인한다.
5. 추후 위치/회전/크기 동기화용 컴포넌트를 붙일 수 있게 구조를 정리한다.

## 공연/AR 맥락 반영
- 이 오브젝트는 공연장 전체 좌표계에 존재하는 하나의 연출체다.
- 관객별로 다른 오브젝트를 생성하지 않고, 동일 오브젝트를 각자의 카메라 시점에서 다르게 보게 하는 구조를 유지한다.

## 가정
- `StageObject_Halo`는 이미 씬에 존재하거나 쉽게 프리팹화할 수 있다.
- 공연장 원점/앵커 보정은 별도 시스템에서 제공되거나 이후 AR 표시 단계와 연결된다.

## 검증 필요
- Scene Object와 Runtime Spawn 중 현재 프로젝트 구조에 더 덜 충돌하는 방식 검토가 필요하다.
- 기존 씬 안의 다른 Fusion 컴포넌트와 소유권 충돌이 없는지 확인이 필요하다.

## 완료 기준
- `StageObject_Halo`가 네트워크 상에서 하나의 권위 오브젝트로 식별된다.
- Host만 생성/소유/갱신하고 클라이언트는 수신만 하는 구조가 정리된다.
- 다음 Task에서 위치/회전/크기 동기화를 구현할 준비가 된다.
