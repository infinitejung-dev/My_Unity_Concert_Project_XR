# Task 06. 3클라이언트 통합 리허설 검증

## 목적
Host PC 1대와 클라이언트 3대 환경에서 `StageObject_Halo` 동기화와 AR 표시를 빠르게 검증할 수 있는 리허설 절차와 디버그 수단을 정리한다.

## 구현 대상
- 세션 상태 확인 HUD 또는 디버그 패널
- `StageObject_Halo` 현재 위치/회전/크기 값 표시
- Host/Client 역할, 접속 수, 마지막 동기화 시점 확인
- 리허설 순서에 맞는 테스트 절차

## 구현 범위
- 복잡한 운영 대시보드가 아니라 프로토타입 검증에 필요한 최소 상태만 표시한다.
- Unity Editor 1대, PC 빌드 1대, 모바일 2~3대 조합 등 현실적인 리허설 구성을 지원한다.
- 결과 기록은 수동 체크리스트 기반으로 충분하다.
- 이 단계에서 큰 구조 변경보다 검증 반복 속도를 높이는 것을 우선한다.

## 구현 단계
1. Host와 Client 모두 현재 역할과 연결 상태를 볼 수 있게 한다.
2. `StageObject_Halo`의 위치/회전/크기 값을 숫자로 확인할 수 있게 한다.
3. 3대 동시 접속, Host 이동/회전/스케일 변경, 모바일 시점 변화 테스트 순서를 정리한다.
4. 접속 중 이탈 및 재접속 상황에서 상태 복구가 되는지 확인한다.
5. 다음 연출 Task로 넘어가기 전에 남은 리스크를 기록한다.

## 공연/AR 맥락 반영
- 공연 리허설처럼 빠르게 세팅하고 바로 검증할 수 있어야 다음 연출 실험 속도가 난다.
- 동일 월드 상태와 관객별 시점 차이를 동시에 확인할 수 있는 최소 도구가 필요하다.

## 가정
- 디버그 UI는 개발용이며, 최종 관객 화면에는 노출하지 않는다.
- 접속 환경은 실제 공연장보다 단순한 프로토타입 네트워크 환경이다.

## 검증 필요
- 모바일 기기 3대 동시 구동 시 발열과 프레임 저하가 체감되는지 확인이 필요하다.
- 네트워크 지연이 커졌을 때 위치/회전/크기 오차 허용 기준을 팀에서 정해야 한다.

## 완료 기준
- 3클라이언트 통합 리허설을 같은 절차로 반복 수행할 수 있다.
- 문제 발생 시 Host, 접속, Transform, AR 표시 중 어디에서 깨졌는지 빠르게 좁힐 수 있다.
- 이후 연출 타이밍, 다중 오브젝트, 관객별 인터랙션 같은 확장 Task로 넘어갈 준비가 된다.
