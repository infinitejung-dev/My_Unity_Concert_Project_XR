# 단계별 수행 절차

## 목적
이 문서는 `Assets/Tasks`의 02~11번 작업으로 구현된 프로토타입 스크립트를 Unity 에디터 씬에 연결하는 절차를 정리한다. 목표는 중앙 마커를 기준으로 한 `Stage Space`, 관객 A/B 시점, 공통 이벤트 타임라인, 3D 오브젝트 Presenter, 디버그 오버레이를 한 씬에서 순서대로 검증하는 것이다.

## 1. 스크립트 컴파일 확인
1. Unity 에디터에서 프로젝트를 연다.
2. Console 창에 C# 컴파일 오류가 없는지 확인한다.
3. `Assets/Scripts/Audience`, `Assets/Scripts/Calibration`, `Assets/Scripts/Events`, `Assets/Scripts/Presentation`, `Assets/Scripts/Prototype` 폴더에 `SUN_` prefix 스크립트가 생성되어 있는지 확인한다.

## 2. 중앙 마커와 StageCoordinateSystem 구성
1. Hierarchy에서 빈 GameObject를 만들고 이름을 `SUN_StageRoot`로 변경한다.
2. `SUN_StageRoot` 아래에 빈 GameObject를 만들고 이름을 `CentralMarker`로 변경한다.
3. `CentralMarker`의 Transform을 Position `(0, 0, 0)`, Rotation `(0, 0, 0)`, Scale `(1, 1, 1)`로 설정한다.
4. `CentralMarker` 아래에 Cube 또는 Cylinder를 만들고 이름을 `CentralMarker_Visual`로 변경한다.
5. `SUN_StageRoot`에 `SUN_StageCoordinateSystem` 컴포넌트를 추가한다.
6. `SUN_StageCoordinateSystem`의 `_stageOrigin`에 `CentralMarker`를 할당한다.
7. `_stageForward`는 `(0, 0, 1)`, `_stageRight`는 `(1, 0, 0)`, `_metersToUnityScale`은 `1`로 둔다.
8. Scene 뷰 Gizmo에서 흰 원점, 빨간 X축, 초록 Y축, 파란 Z축, 청록 무대 방향선이 보이는지 확인한다.

## 3. 관객 프로필 A/B 생성
1. Project 창에서 `Assets` 아래에 `Profiles` 폴더를 만든다.
2. `Assets/Profiles`에서 우클릭 후 `Create > SUN > Prototype > Audience Profile`을 선택한다.
3. 첫 번째 에셋 이름을 `SUN_AudienceProfile_A`로 변경한다.
4. `SUN_AudienceProfile_A` 값을 다음 예시로 입력한다.
   - `_audienceId`: `Audience_A`
   - `_seatStagePositionMeters`: `(-2, 0, -4)`
   - `_eyeHeightMeters`: `1.6`
   - `_deviceOffsetMeters`: `(0, 0, 0)`
   - `_fieldOfViewDegrees`: `45`
5. 같은 방식으로 `SUN_AudienceProfile_B`를 만들고 다음 예시값을 입력한다.
   - `_audienceId`: `Audience_B`
   - `_seatStagePositionMeters`: `(2, 0, -5)`
   - `_eyeHeightMeters`: `1.55`
   - `_deviceOffsetMeters`: `(0.02, 0, 0.02)`
   - `_fieldOfViewDegrees`: `50`

## 4. 관객 Rig와 카메라 구성
1. Hierarchy에 빈 GameObject `Audience_A_Rig`를 만든다.
2. `Audience_A_Rig` 아래에 Camera를 만들고 이름을 `Audience_A_Camera`로 변경한다.
3. `Audience_A_Rig`에 `SUN_AudienceRig` 컴포넌트를 추가한다.
4. `_profile`에 `SUN_AudienceProfile_A`, `_cameraRoot`에 `Audience_A_Camera`, `_coordinateSystem`에 `SUN_StageRoot`의 `SUN_StageCoordinateSystem`을 할당한다.
5. 통합 ViewController 검증에서는 `_applyToCameraRoot`를 꺼 둔다. Rig 단독 위치 검증만 할 때는 켜서 카메라 루트 이동을 확인한다.
6. `Audience_A_Camera`에 `SUN_AudienceViewController` 컴포넌트를 추가한다.
7. `_audienceCamera`에 `Audience_A_Camera`, `_audienceRig`에 `Audience_A_Rig`의 `SUN_AudienceRig`를 할당한다.
8. 같은 방식으로 `Audience_B_Rig`, `Audience_B_Camera`를 만들고 B 프로필을 연결한다.
9. 두 카메라를 동시에 비교하려면 `Audience_A_Camera`의 Viewport Rect를 `(0, 0, 0.5, 1)`, `Audience_B_Camera`의 Viewport Rect를 `(0.5, 0, 0.5, 1)`로 설정한다.
10. Audio Listener가 두 개 이상 있으면 하나만 남기거나 비활성화한다.

## 5. 중앙 마커 캘리브레이터 구성
1. Hierarchy에 빈 GameObject `SUN_Calibration`을 만든다.
2. `SUN_Calibration`에 `SUN_CentralMarkerCalibrator` 컴포넌트를 추가한다.
3. `_markerVisual`에 `CentralMarker_Visual`, `_coordinateSystem`에 `SUN_StageRoot`의 `SUN_StageCoordinateSystem`을 할당한다.
4. `_useManualCalibration`을 켠다.
5. 첫 검증에서는 `_manualMarkerPositionMeters`를 `(0, 0, 0)`, `_manualMarkerEulerDegrees`를 `(0, 0, 0)`으로 둔다.
6. `Audience_A_Camera`, `Audience_B_Camera`의 `SUN_AudienceViewController`에 `_calibrator`로 `SUN_Calibration`의 캘리브레이터를 할당한다.
7. 수동 보정 영향 검증 시 `_manualMarkerEulerDegrees`의 Y값을 `10` 또는 `-10`으로 바꾼 뒤 다시 Calibrate를 실행해 카메라와 오브젝트가 같은 기준으로 회전하는지 확인한다.

## 6. 이벤트 타임라인 구성
1. Hierarchy에 빈 GameObject `SUN_EventTimeline`을 만든다.
2. `SUN_EventTimeline`에 `SUN_StageEventTimeline` 컴포넌트를 추가한다.
3. `_playOnStart`는 초기 검증에서는 꺼 두고, 수동 Play가 확인된 뒤 필요하면 켠다.
4. `_events`의 Size를 `1` 이상으로 설정한다.
5. 첫 이벤트 예시는 다음처럼 입력한다.
   - `EventId`: `Event_Halo_Rise`
   - `ObjectId`: `StageObject_Halo`
   - `StartTimeSeconds`: `0`
   - `DurationSeconds`: `5`
   - `StartPositionMeters`: `(0, 1, 2)`
   - `EndPositionMeters`: `(0, 3, 5)`
   - `StartEulerDegrees`: `(0, 0, 0)`
   - `EndEulerDegrees`: `(0, 180, 0)`
   - `StartScale`: `(0.5, 0.5, 0.5)`
   - `EndScale`: `(1.5, 1.5, 1.5)`
   - `IsVisible`: `true`

## 7. AR 오브젝트 Presenter 구성
1. Hierarchy에 Sphere, Cube, 또는 사용할 3D 오브젝트를 만들고 이름을 `StageObject_Halo`로 변경한다.
2. `StageObject_Halo`에 `SUN_ARObjectPresenter` 컴포넌트를 추가한다.
3. `_objectId`를 `StageObject_Halo`로 입력한다.
4. `_targetObject`에 `StageObject_Halo`의 Transform을 할당한다.
5. `_renderers`에는 `StageObject_Halo`의 Renderer를 할당한다. 비워 둬도 Awake에서 하위 Renderer를 캐시할 수 있다.
6. `_coordinateSystem`에 `SUN_StageRoot`의 `SUN_StageCoordinateSystem`을 할당한다.
7. `_timeline`에 `SUN_EventTimeline`의 `SUN_StageEventTimeline`을 할당한다.
8. `_calibrator`에 `SUN_Calibration`의 `SUN_CentralMarkerCalibrator`를 할당한다.
9. 오브젝트가 카메라를 강제로 바라보지 않고, 타임라인의 Stage Space 위치/회전/스케일만 따르는지 확인한다.

## 8. PrototypeSceneController 구성
1. Hierarchy에 빈 GameObject `SUN_PrototypeSceneController`를 만든다.
2. `SUN_PrototypeSceneController` 컴포넌트를 추가한다.
3. `_coordinateSystem`, `_calibrator`, `_timeline`에 앞서 만든 컴포넌트들을 할당한다.
4. `_audienceRigs` 배열 Size를 `2`로 만들고 `Audience_A_Rig`, `Audience_B_Rig`를 할당한다.
5. `_viewControllers` 배열 Size를 `2`로 만들고 `Audience_A_Camera`, `Audience_B_Camera`의 ViewController를 할당한다.
6. `_objectPresenters` 배열에 `StageObject_Halo`의 Presenter를 할당한다.
7. 키보드 수동 제어 기본값을 확인한다.
   - `Space`: Timeline Play/Pause
   - `R`: Prototype Reset
   - `C`: Manual Calibrate
   - `T`: Rescan Marker
   - `Backspace`: Calibration Reset
   - `V`: Debug Overlay On/Off
   - `1`, `2`: 제어할 관객 선택
   - `W/A/S/D`: 선택 관객 좌석 오프셋 이동
   - `Q/E`: 선택 관객 yaw 조정
   - `Z/X`: 선택 관객 pitch 조정
   - `-`, `=`: 선택 관객 FOV 조정

## 9. DebugOverlay 구성
1. Hierarchy에 UI Canvas를 만들고 이름을 `SUN_DebugCanvas`로 변경한다.
2. `SUN_DebugCanvas` 아래에 Text UI를 3개 만들고 각각 `StatusText`, `WarningText`, `DetailsText`로 변경한다.
3. 세 Text를 화면 좌상단에 세로로 배치하고, 긴 좌표 문자열이 잘리지 않도록 Width를 충분히 넓힌다.

4. Hierarchy에 빈 GameObject `SUN_DebugOverlay`를 만든다.
5. `SUN_DebugOverlay`에 `SUN_PrototypeDebugOverlay` 컴포넌트를 추가한다.
6. `_displayRoot`에 `SUN_DebugCanvas`, `_statusText`에 `StatusText`, `_warningText`에 `WarningText`, `_detailsText`에 `DetailsText`를 할당한다.
7. `_coordinateSystem`, `_calibrator`, `_timeline`, `_audienceRigs`, `_viewControllers`, `_objectPresenters`를 수동으로 할당하거나, `SUN_PrototypeSceneController`의 `_debugOverlay`에 `SUN_DebugOverlay`를 할당해 시작 시 자동 바인딩되게 한다.

8. Play 모드에서 화면 좌상단에 상태, 누락 참조, 캘리브레이션 상태, 타임라인 시간, 관객 눈 위치, 오브젝트 좌표가 표시되는지 확인한다.
9.  Scene 뷰에서 Stage 축, 관객 몸/눈 위치, Presenter 위치 Gizmo가 보이는지 확인한다.

## 10. 통합 검증 순서
1. Play 모드를 시작한다.
2. Console에 누락 참조 경고가 있는지 확인한다. 의도한 미연결 항목이 아니면 Inspector 연결을 수정한다.
3. `C`를 눌러 수동 중앙 마커 캘리브레이션을 실행한다.
4. DebugOverlay의 Calibration 상태가 `Ready`이고 마지막 보정 시간이 갱신되는지 확인한다.
5. `Space`를 눌러 타임라인을 재생한다.
6. `StageObject_Halo`가 `Stage Space` 이벤트 값에 따라 위치, 회전, 스케일을 바꾸는지 확인한다.
7. 좌우 분할 카메라 화면에서 같은 오브젝트가 관객 A/B 위치 차이 때문에 서로 다른 각도와 거리감으로 보이는지 확인한다.
8. `1` 또는 `2`로 관객을 선택하고 `W/A/S/D`, `Q/E`, `Z/X`, `-`, `=`를 눌러 위치, 시선, FOV 변화를 확인한다.
9. `Backspace`로 캘리브레이션만 초기화하고, `T`로 Reset 후 즉시 재보정되는지 확인한다.
10. `R`을 눌러 Reset 후 관객 오프셋, 시선, FOV, 타임라인, 캘리브레이션이 초기화되는지 확인한다.
11. `_manualMarkerEulerDegrees` 또는 `_manualMarkerPositionMeters`를 변경하고 다시 `C`를 눌러 카메라와 오브젝트가 같은 보정 기준으로 움직이는지 확인한다.

## 11. 이번 프로토타입에서 코드로 지원되는 항목
1. 관객 프로필 ScriptableObject 생성 메뉴.
2. 관객 몸 위치와 눈 위치의 Stage Space 계산.
3. Stage Space와 Unity World 변환 API 사용.
4. 수동 중앙 마커 캘리브레이션과 TrackingLost 상태 표시.
5. 관객 카메라 위치, 회전, FOV clamp.
6. 공통 타임라인 Play/Pause/Reset과 오브젝트 상태 평가.
7. Presenter의 Transform/Renderer 적용과 표시 상태 분리.
8. 씬 컨트롤러의 수동 키보드 제어.
9. 디버그 오버레이 화면 텍스트와 Scene Gizmo 표시.

## 12. Unity 에디터에서 남은 수동 검증 항목
1. 관객 A/B 프로필 에셋을 실제로 생성하고 좌석, 눈 높이, FOV 값을 입력한다.
2. 관객 A/B Rig와 카메라를 실제 씬에 연결한다.
3. 중앙 마커 Visual과 StageCoordinateSystem 축 방향을 Scene 뷰에서 확인한다.
4. 테스트 3D 오브젝트와 Timeline 이벤트 값을 실제 Inspector에 입력한다.
5. 좌우 분할 카메라 또는 다른 비교 뷰를 구성해 관객별 원근 차이를 확인한다.
6. 시야 밖 판정과 이벤트 데이터 오류의 오버레이 직접 표시는 필요 시 후속 작업으로 추가한다.
