# Task 02. 3클라이언트 세션 접속 경로 구성

## 목적
Host로 열린 Fusion 세션에 `Host PC 1대 + 클라이언트 3대` 구성이 안정적으로 진입할 수 있도록 접속 흐름과 제한 정책을 정리한다.

## 구현 대상
- 세션 참가용 클라이언트 시작 진입점
- 3명 동시 접속 제한
- 접속 상태 표시와 실패 안내
- Editor, PC 빌드, 스마트폰 빌드 중 최소 3대 조합 테스트 흐름

## 구현 범위
- 클라이언트는 Host가 만든 동일 세션명 또는 동일 룸 선택 로직으로 접속한다.
- 클라이언트 3명을 초과하는 접속은 차단되거나 명확한 실패 상태를 보여준다.
- 관객별 개별 아바타 동기화는 이번 범위에서 제외한다.
- 연결 성공 여부, 현재 접속 수, 내 역할(Host/Client) 정도만 표시하면 충분하다.

## 구현 단계
1. Host 진입과 별도로 Client 진입 경로를 분리한다.
2. 세션 이름 또는 연결 키 입력 방식을 최소화해 리허설 중 반복 접속이 쉽도록 한다.
3. 최대 3명 제한을 Fusion 설정과 세션 흐름 모두에서 확인한다.
4. 클라이언트가 접속 후 현재 세션 상태를 확인할 수 있는 간단한 UI 또는 로그를 추가한다.
5. 동일 시간대에 3대가 접속한 뒤, 4번째 접속 시도를 어떻게 처리할지 검증한다.

## 공연/AR 맥락 반영
- 각 관객은 같은 공연장 월드 오브젝트를 보게 되지만, 네트워크 단계에서는 우선 동일 세션 소속인지가 핵심이다.
- 관객별 실제 시야 차이와 카메라 차이는 다음 AR 표시 Task에서 다룬다.

## 가정
- 스마트폰 클라이언트는 Wi-Fi 환경으로 Host PC와 같은 세션에 들어올 수 있다.
- 프로토타입 단계에서는 계정 로그인 없이 세션명 기반 접속으로 충분하다.

## 검증 필요
- 동일 플랫폼 3대와 혼합 플랫폼 3대에서 접속 안정성 차이가 있는지 확인이 필요하다.
- 모바일에서 세션 재접속 시 이전 세션 정리가 자동으로 되는지 확인이 필요하다.

## 완료 기준
- Host PC 1대 외에 클라이언트 3대가 같은 세션에 들어오는 흐름이 재현 가능하다.
- 접속 초과, 접속 실패, 세션 미존재 상태를 구분해서 볼 수 있다.
