# Checklist 06. 3클라이언트 통합 리허설 검증

## 구현 확인
- [ ] Host/Client 역할과 접속 상태를 볼 수 있는 HUD 또는 디버그 패널이 있다.
- [ ] `StageObject_Halo` 위치/회전/크기 값을 숫자로 확인할 수 있다.
- [ ] 3클라이언트 리허설 절차가 문서 또는 UI 흐름으로 정리되어 있다.

## 통합 동작 검증
- [ ] Host PC 1대와 클라이언트 3대 조합으로 같은 세션에 접속할 수 있다.
- [ ] Host에서 `StageObject_Halo`를 조작하면 3대 클라이언트에 모두 반영된다.
- [ ] 3대 클라이언트가 각자 다른 시점에서 같은 오브젝트를 본다.
- [ ] 클라이언트 1대가 이탈 후 재접속해도 최신 상태를 다시 받는다.
- [ ] 리허설 중 문제 발생 시 접속 문제인지, 동기화 문제인지, AR 표시 문제인지 빠르게 구분할 수 있다.

## 프로토타입 맥락 검증
- [ ] 이번 범위가 서버 설치/구성 및 단일 오브젝트 동기화 검증에 집중되어 있다.
- [ ] 상용 운영 기능 대신 핵심 경험 검증 중심으로 유지된다.

## 완료 판정
- [ ] 다음 연출 확장 작업 전에 네트워크/AR 기본기가 검증됐다.
