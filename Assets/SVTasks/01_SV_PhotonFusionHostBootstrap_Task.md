# Task 01. Photon Fusion 호스트 부트스트랩 구성

## 목적
개인 PC를 Fusion Host로 사용하기 위한 최소 부트스트랩을 만든다. 이미 프로젝트에 Fusion 패키지가 들어와 있다면 재설치보다 설정 점검과 누락 보완을 우선한다.

## 구현 대상
- Photon Fusion 패키지 설치 상태 확인
- `NetworkProjectConfig` 및 App 설정 점검
- 개인 PC에서 실행되는 `NetworkRunner` 시작 진입점
- 프로토타입용 세션명, 게임 모드, 기본 네트워크 씬 연결

## 구현 범위
- Host 또는 Server/Client 중 현재 프로토타입에 가장 단순한 `Host` 모드를 우선 채택한다.
- 개인 PC 1대가 세션 권위를 가진다.
- 최대 동시 접속 수는 이후 Task에서 실제 검증하지만, 이 단계에서 설정값 기준은 `Host PC 1대 + 클라이언트 3대`를 수용하도록 맞춘다.
- `StageObject_Halo` 동기화 로직은 아직 넣지 않고, 세션이 열리고 유지되는 것까지만 다룬다.

## 구현 단계
1. 프로젝트 안의 Fusion 설치 상태와 버전 관련 설정 파일 존재 여부를 확인한다.
2. `NetworkProjectConfig`에서 세션 기본값과 플레이어 수 제한을 프로토타입 기준으로 정리한다.
3. 호스트 시작을 담당할 부트스트랩 컴포넌트 또는 진입 흐름을 만든다.
4. 샘플 씬 또는 프로토타입 씬에서 Play 시 Host 세션이 열리는지 확인한다.
5. 연결 실패 시 원인을 볼 수 있는 최소 로그를 남긴다.

## 공연/AR 맥락 반영
- 공연장 좌표계에서 권위 있는 월드 기준은 Host가 가진다.
- 관객별 시점 차이는 이후 클라이언트 카메라에서 처리하고, 이 Task에서는 동일한 세션에 접속할 수 있는 기반만 마련한다.

## 가정
- 로컬 네트워크 또는 동일 인터넷 환경에서 우선 검증한다.
- 방화벽, 포트, Photon Cloud 인증 정보는 이미 준비되어 있거나 프로젝트에서 확인 가능하다.

## 검증 필요
- 현재 프로젝트가 Cloud 연결형인지, 완전 로컬 전용 테스트인지 확인이 필요하다.
- 스마트폰 빌드와 Editor 간 접속 조합에서 추가 네트워크 예외 처리가 필요한지 확인이 필요하다.

## 완료 기준
- 개인 PC에서 Host 세션을 시작할 수 있다.
- 세션 시작 성공/실패 로그가 Unity Console에서 식별 가능하다.
- 다음 Task에서 3클라이언트 접속 흐름을 붙일 수 있는 상태가 된다.
