# Checklist 02. 3클라이언트 세션 접속 검증

## 구현 확인
- [ ] Host와 Client 시작 경로가 분리되어 있다.
- [ ] 세션명 또는 접속 키 입력 방식이 정리되어 있다.
- [ ] 최대 3명 제한 정책이 문서 또는 코드에 반영되어 있다.
- [ ] 접속 상태를 볼 수 있는 최소 UI 또는 로그가 있다.

## 동작 검증
- [ ] 클라이언트 1명이 Host 세션에 정상 접속한다.
- [ ] 클라이언트 2명이 동시에 접속해도 세션이 유지된다.
- [ ] 클라이언트 3명이 동시에 접속해도 세션이 유지된다.
- [ ] 4번째 접속 시도 시 허용되지 않거나 명확한 실패 상태를 보여준다.
- [ ] 접속 실패, 세션 미존재, 세션 가득 참 상태를 구분할 수 있다.

## 프로토타입 맥락 검증
- [ ] 같은 공연장 연출을 볼 동일 세션 구성원이라는 개념이 유지된다.
- [ ] 관객별 개별 오브젝트를 만들지 않고 공용 세션에 접속하는 구조다.

## 완료 판정
- [ ] 최소 3대 클라이언트 접속 흐름이 반복 가능하다.
